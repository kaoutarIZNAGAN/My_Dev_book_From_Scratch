# Pyramide d'apprentissage Backend

## Base 1 : Langage et logique de programmation:

### Objectif :

1. créer un projet backend propre
2. comprendre ton propre code
3. ne pas avoir peur de casser et réparer

Culture générale:
- Comprendre le codage

Langage JS :

- les catégories en js
- Fonctions
- Async / await
- Promises
- Callbacks
- Arrays methodes
- String méthodes
- Gestion des erreurs (try/catch)
- Compréhension du event loop
- POO
- DRY/KISS
- Commandes terminal de base

Méthodologie de Travail :

- Création projet, modification, nettoyage des données)
- Organisation des fichiers
- Lisibilité du code

Git :

- Markdown pour la documentation git
- Comprendre l'architecture Git ( Repository, Working directory, staging, commit)
- Historique
- Commandes principales Push / pull ..
- Gestion des branches.
- Versionning

## Base 2 : API et serveurs

### Objectif :

1. créer une API fonctionnelle
2. comprendre une requête HTTP de bout en bout
3. expliquer ce que fait ton backend

Backend pur :

- Node.js
- Express framework
- Middlewares
- erreur handling global
- Conception d’API: voir la video : How to Design APIs Like a Senior Engineer (REST, GraphQL, Auth, Security)
- Swagger/OpenAPI pour documenter l'API
- Protocoles REST
- protocole HTTP
- JSON
- Postman
- Debug (console + navigateur en inspection )

Architecture :

- Séparation routes, controllers, services..
- Logique métier
- Dossiers propres

## Base 3 : Base de données (structure et fiabilité)

### Objectif :

1. tes données sont protégées
2. ton schéma peut évoluer
3. tu comprends l’impact d’un mauvais modèle

BDD :

- Modélisation des données
- Relations
- Requêtes
- Contraintes
- Indexation
- Migrations
- CRUD complet

ORM :

- ORM indispensable en JS
- Modèles
- Relations ORM
- Différences DEV / PROD

## Base 4 : AUthentification et Sécurité des données

### Objectif :

1. ton application peut être exposée au public
2. tu comprends les risques
3. tu sais les éviter

Authentification : video : Authentication Explained: When to Use Basic, Bearer, OAuth2, JWT & SSO

- JWT
- Sessions
- Cookies
- Protection des routes
- Gestion des permissions

Sécurité backend :

- Hash des mots de passe
- Rate limiting
- CORS
- Validation des entrées (Zod / Joi)
- Protection brute-force
- Headers de sécurité (Helmet)
- Logs sans données sensibles

### Objectif :

Tests :

- Tests unitaires
